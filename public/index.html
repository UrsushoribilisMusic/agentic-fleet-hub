Everything you need to know before deploying your fleet.

Instead of API keys with per-token billing, we configure agents to authenticate using OAuth with standard model subscriptions (e.g. Claude Pro). This turns unpredictable, per-token costs into a flat, predictable monthly fee — around $20 per agent.

We also implement structural guardrails to prevent agents from going down rabbit holes: task scoping rules, timeout policies, and Kanban-enforced boundaries that keep every agent focused on its assigned ticket and nothing else.

Two reasons: pace and resilience. The AI landscape is moving fast — a model that leads today may be surpassed next month. Being model-agnostic means you always have access to the best tool for each job without rebuilding your infrastructure.

It also eliminates single points of failure. If one provider has an outage or hits a rate limit, the fleet keeps running. Each model also brings genuine strengths — Claude for code precision, Gemini for large-context tasks, Codex for rapid generation.

Far beyond coding. Agents can be assigned roles including:

  • Research & information gathering — summarizing documents, monitoring competitors, pulling external data
  • QA & API architecture — prototyping endpoints, maintaining test suites, CI/CD integration
  • Infrastructure & automation — cloud management, secrets handling, cross-agent synchronization
  • Commerce & operations — webhook orchestration, order routing, marketplace logic

During the Fleet Deployment Intensive, we map your specific workflows to agent roles tailored to your team's actual needs.

Open-weight models are AI models whose parameters are publicly released — meaning you download and run them on your own hardware, with zero data sent to a third party. There is no per-token billing and no external dependency.

We particularly recommend Mistral (French) and Apertus (Swiss-led) for European teams prioritizing data sovereignty. Models from other ecosystems — such as Qwen from China — are also available and can be installed in minutes using Ollama, a lightweight tool that lets you run and switch between open-weight models locally with a single command.

GitHub is already the center of every engineering team's infrastructure. We leverage it for everything the fleet needs:

  • Token & credential management — model API tokens and secrets stored securely via GitHub Secrets
  • Kanban & ticketing — agents pull tasks from GitHub Projects boards and update status asynchronously
  • Source control — every agent configuration, memory file, and rule set is version-controlled
  • Memory storage — MD files live in the repo, linked and accessible to all agents in the fleet

Your fleet lives where your code already lives — no new tooling to adopt.

The Fleet Hub dashboard gives you a live command center for your entire operation:

  • Agent Team view — each agent's role, deployment environment (local, cloud, hybrid), and capabilities at a glance
  • Memory Tree — your centralized knowledge base with global rules (commits, Kanban protocols, secrets strategy) and per-project context files, all linked directly to GitHub
  • Multi-project management — manage several engineering projects simultaneously, each with its own context, agent assignments, and Kanban board
  • Preconfigured rule sets — global rules that govern all agents across the org, plus per-project overrides for specialized workflows
  • Daily Standups — automated session logs from each agent, documenting what was completed, what's in progress, and any blockers — giving you a full audit trail without micromanaging
