# EU AI Act Mapping Schema for Agentegra ATF

## 1. Overview
This document defines the metadata schema for mapping Agentegra ATF wiki pages and ledger entries to the requirements of the EU AI Act (specifically for high-risk and transparency obligations).

## 2. Metadata Frontmatter Schema
All wiki pages generated by the ATF pipeline should include the following frontmatter fields:

```markdown
---
eu_ai_act_obligations:
  - transparency
  - traceability
  - record-keeping
  - human_oversight
compliance_theme: "Transparency / Traceability"
provenance:
  - "path/to/source/file.py:L100-L150"
  - "mexico_log_2026-04-09.log#E-001"
---
```

## 3. Obligations Mapping Table

| Obligation | Description | Wiki/Ledger Implementation |
| :--- | :--- | :--- |
| **Transparency** | Ensure users are aware they are interacting with an AI. | Handled via [[Narration]] and clearly labeled proof-of-work. |
| **Traceability** | Maintain logs and documentation of AI system logic and decisions. | Handled via the [[JobOrchestration]] wiki and Mexico production logs. |
| **Record-keeping** | Automatic logging of system activity and performance. | Handled via the `orders.json` ledger and `task_events` telemetry. |
| **Human Oversight** | Ensure a human can intervene or override AI decisions. | Handled via the Salesman Dashboard and the manual [[HardwareInterface]] controls. |
