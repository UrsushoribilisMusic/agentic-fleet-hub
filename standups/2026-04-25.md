# Clau — 2026-04-25 (session 2)

## Task: PC-030 [P1]: Person card — backlink integration (addep7b9)

**Status**: done

### What was implemented

The infrastructure (LinkParser, LinkService, person_links table, PersonDetailView "Mentioned In" section, wikilink autocomplete in AddCardSheet + CardDetailView) was already in place from prior sessions. Two bugs blocked the feature from working end-to-end:

**Bug 1 — "Mentioned In" always empty (PersonDetailView)**
`loadLinkedItems()` queried `person_links(sourceType: "card")` twice using identical queries, then subtracted one set from the other — resulting in an always-empty "Mentioned In" section. Fixed by removing the redundant `linkedCards` state and query; `fetchCardsByPersonLink()` is now used directly to populate `mentionedInCards`.

**Bug 2 — Stale backlinks persist after edit (LinkService)**
`LinkService.sync()` only ever appended to `person_links` (via `insertPersonLink` with `ON CONFLICT DO NOTHING`). Removing a `[[PersonName]]` from a card never deleted the old link. Fixed by calling `deletePersonLinks(sourceType: "card", sourceId: noteId)` before re-inserting, mirroring how `syncEdges` handles generic backlinks.
