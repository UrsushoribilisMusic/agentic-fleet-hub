### PrivateCore iOS — bug fixes (commit `dd49ba8`)

Four bugs fixed from on-device testing session:

1. **DB migration** — `migrateSchema()` added to `SQLiteStore.ensureOpen()`. Uses `PRAGMA table_info` to detect missing columns and `ALTER TABLE ADD COLUMN` to add them. Fixes `no such column: template_hash` on photos (Ask crash) and silent insert/fetch failures on cards (`embedding_id`, `parent_id` missing on upgraded installs). Root cause: `CREATE TABLE IF NOT EXISTS` never evolves existing tables.
2. **Capture crash** — `capturedImage = image` in AVFoundation delegate callback was mutating `@State` off main thread. Fixed with `DispatchQueue.main.async`.
3. **Settings download UI** — `SettingsView` was a complete static stub. Replaced with real `ModelRow` per model: Ready (green checkmark) / Download button / progress bar / Failed+retry. `ModelManager.availableModels` and `isModelDownloaded` marked `nonisolated` for sync SwiftUI access.
4. **Add Sample Cards** — root cause same as #1 (missing columns caused silent fetch failures after insert).
