April 10 had zero ads — root cause of near-zero views on boosted videos.

### Diagnosis

- Campaign `TCR DG | danse-macabre--vallenato | 2026-04-10 18:14 UTC` was ENABLED
  with 3 ad groups but **no ads**. Delivered nothing for 4 days.

  the account. Ads were silently rejected.

  `businessName`, `longHeadlines`. All required by v20 API.
- `get_song_for_recommendation()` (executor) and `choose_song_for_recommendation()`
  (campaign manager) did not filter boosted songs — same song recommended every day.

Tried VIDEO / TARGET_CPV / VIDEO_TRUE_VIEW_IN_DISPLAY for better subscription
and watch-time performance. **Blocked: account does not have Video API write access.**
`MUTATE_NOT_ALLOWED` on all VIDEO-type campaign, ad group, and ad mutations.
The existing "Music videos promotion April" campaign was created via UI only.
Reverted to Demand Gen (which the API allows on this account).

### Fixes applied (committed to `~/fleet`)

**`tcr_ads_executor.py`:**
- `get_or_create_logo_asset()`: generates and uploads a 128×128 PNG if no asset
  exists (pure stdlib — no PIL required).

  `ad.finalUrls`, `businessName` as AdTextAsset, `longHeadlines` (1 min),

## Clau session — ATF local server build (~18:00 UTC)

**Scope:** Wire the ATF "Local Reasoning Session" to the real Apertus model running on the Mac Mini, preserving the public cloud version unchanged.

### Design decision

Two separate pages:
- `ATF/index.html` — public cloud version, untouched, mock rotating examples only
- `ATF/index_local.html` — local version, talks to `atf_local_server.py` on port 8771

### Files created / modified

**`ATF/tools/atf_local_server.py`** (new):
- Python stdlib `http.server`, no dependencies beyond the existing ATF tools
- Serves all ATF static files at `http://localhost:8771/`
- `POST /api/qa` — runs the real `atf_qa` pipeline, returns `{"answer", "backend", "model", "sources"}`
- `GET /api/models` — returns available Ollama models + auto-selected model name
- Port 8770 was occupied by macOS `sharingd`; defaulted to 8771

**`ATF/tools/runtime_adapter.py`**:
- Added `query_with_model()` — same as `query()` but returns `(text, model_name)` so callers can surface which Ollama model actually answered. `query()` now delegates to it.

**`ATF/index_local.html`** (new, copied from `index.html` then modified):
- Model dropdown populated from `/api/models` on load; auto-selected model pre-highlighted; user can pick any pulled model before querying
- Answer box split into main answer (normal text) and sources (small italic, separated by a rule)
- Backend label shows the real model tag (e.g. `apertus:8b-instruct-2509-q4_k_m`) instead of `runtime_adapter`
- Voice input via Web Speech API: pulsing red indicator while listening, live interim transcript in the textarea, auto-submits 0.9s after speech ends, graceful error if mic is denied or browser unsupported

### To run

```bash
python3 ATF/tools/atf_local_server.py
# open http://localhost:8771/index_local.html
```
