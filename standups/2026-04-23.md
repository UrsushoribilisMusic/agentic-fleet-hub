## PC-046: Wire real MLX Swift inference into MLXEngine

**Status**: approved (peer review by Claude Code 2026-04-23 — Teddy session)

### What shipped

**MLXEngine.swift** — complete rewrite, all mocks removed:
- Imports `MLXLLM`, `MLXVLM`, `MLXLMCommon` from mlx-swift-lm 2.31.3
- `load(config:)` calls `loadModelContainer(directory:)` — the top-level registry-walking function; both MLXLLM and MLXVLM TrampolineModelFactory classes self-register via `NSClassFromString`, so the correct backend is selected automatically from `config.json`
- `generate(prompt:systemPrompt:image:)` creates a single-turn `ChatSession` and calls `streamResponse(to:image:)` → `AsyncThrowingStream<String, Error>`, bridged into the existing `AsyncStream<String>` interface

- Actor isolation: container captured before entering `Task { }` (Swift 6 safe)
- Error surface: missing container yields "Go to Settings → Models" rather than crashing

**ModelRouter.swift**:
- Throws `MLXEngineError.modelNotDownloaded` if model not on-disk (callers surface as "download in Settings")
- Hot-swap: `unload()` → `load(config:)` sequence

**ModelManager.swift**:
- SmolLM2 135M: `isBundled: true → false` (was never in bundle)
- Renamed `ModelConfiguration → PCModelConfiguration` (avoid collision with `MLXLMCommon.ModelConfiguration`)

**ChatService.swift**:
- Removed manual `<|system|>...<|user|>...<|assistant|>` formatting — caused double-templating; now passes raw query + systemPrompt to `MLXEngine.generate()` and lets ChatSession apply the model's Jinja template
