**Branch:** `task/atf9-cli-qa`  
**Status:** complete → peer_review

### What was built

**Parser** (`ATF/parsers/mexico_log_parser.py`):
- Parses Mexico wood-marking logs into normalized JSONL event streams
- Extracted 50,878 events from `bob_ross.log` sample
- Identified 26 event categories: GCODE, DRAWING, PEN, JOB, NARRATION, VOICE, AI TOKENS, SKETCH, CHECK, PORT, OBS, CAFFEINATE, STOP, ERROR, etc.

**Normalized Schema** (documented in `ATF/parsers/README.md`):
- Core: `event_id`, `timestamp` (ISO + epoch), `event_type`, `event_category`, `event_action`
- Provenance: `source_file`, `line_number`, `raw_line` (preserved exactly)
- Details: event-specific fields (e.g., `move_current/move_total` for DRAWING, `input/output_tokens` for AI TOKENS)
- Metadata: `parsed_at`, `parser_version`

**Output** (`ATF/artifacts/ledger/mexico_events.jsonl`):
- 50,878 parsed events from bob_ross.log
- Category breakdown: 44,806 GCODE (motion), 3,959 DRAWING, 1,272 PEN, 111 NARRATION, 103 JOB, 69 VOICE, 64 CAFFEINATE, 57 MARKERS, 36 SKETCH, 27 PORT, 20 UNKNOWN, 17 AI, 12 BUSY, 8 ERROR, 4 APPROVE, 3 CHECK/SUBPROCESS, 2 LOCK, 1 DRY
- Time range: 2026-03-19T12:44:47Z to 2026-03-30T20:26:04Z

### Implementation Notes
- Handles malformed Unicode (U+FFFD replacement characters) in raw Windows logs
- Uses MD5-based event IDs for dedup: `hash(source_file:line_number:timestamp)`
- Extracts structured details from each event type with type-aware parsing
- No external dependencies (stdlib only, Python 3.9+)
