# Clau — 2026-04-11 (ATF-11)

## Ticket
**#134 [ATF-11] Optional voice shell with Whisper and Voxtral**

## Work done

Created `ATF/tools/atf_voice.py` — the optional Layer 4 voice shell as
specified in the ATF architecture contract.

### Design decisions

**STT (speech-to-text)**
- Backend: Whisper via `openai-whisper` or `faster-whisper` (auto-detected at runtime).
- `faster-whisper` is tried first — ~4× faster on Apple Silicon (int8 CPU quantisation).
- Default model: `base.en` — good accuracy/latency balance for English commands.
  Override via `ATF_WHISPER_MODEL` env var.

**TTS (text-to-speech) — priority order**
1. **Voxtral** (`voxtral-mini-2507` via Mistral API) — matches the spec; requires
   `MISTRAL_API_KEY` + `pip install mistralai`.  Called in text-in/audio-out mode.
2. **Kokoro** — local neural TTS, no API key, ~100–300 ms on M-series.
3. **macOS `say`** — zero-install on Darwin, ~50 ms.
4. **espeak** — Linux fallback.
5. **PrintTTS** — last resort; prints answer to stdout if no audio output found.

**Latency budget (M2 Mac Mini, Whisper base.en)**
- STT: 0.8–1.5 s  |  QA (Apertus via ollama): 2–10 s  |  TTS (say): ~50 ms
- Round-trip target: **< 15 s** for most queries.

**Modularity contract**
- `atf_voice.py` calls `atf_qa.py` as a subprocess — no shared imports.
- Core toolchain (atf_qa.py, runtime_adapter.py) does not import this file.
- Disabling voice = simply don't invoke `atf_voice.py`. No config changes needed.

**CLI modes**
- `python3 ATF/tools/atf_voice.py`                 → interactive REPL
- `python3 ATF/tools/atf_voice.py --once`          → single utterance, then exit
- `python3 ATF/tools/atf_voice.py --text "query"`  → TTS-only, no mic required
- `python3 ATF/tools/atf_voice.py --check`         → backend detection report

## Blockers

cleanly to macOS `say` without one. No critical-path work is blocked.
